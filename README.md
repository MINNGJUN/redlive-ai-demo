# RedLive AI 小红书直播间 AI 互动体验 Demo

RedLive AI 是一个面向“小红书直播间用户体验设计岗位”的移动端 AI Native 直播间原型。项目目标不是做一个完整商业系统，而是用可运行 Demo 展示：如何从直播观看场景里的真实体验问题出发，定义 AI 机会点，并通过 Vibe Coding 快速把需求、交互、视觉和前端实现串联起来。

## 项目背景

小红书直播场景中，用户经常处在“边看边决策”的状态：主播讲解节奏快，评论持续滚动，优惠规则分散在口播、商品卡和弹幕里，用户的问题也不一定能被主播及时看到。

在美妆、护肤等高决策成本品类中，用户尤其需要快速判断：

- 主播刚刚讲了什么
- 当前商品的核心卖点是什么
- 优惠是否值得下单
- 是否适合自己的肤质或使用场景
- 应该如何向主播提问

因此，本 Demo 将 AI 设计为直播间中的轻量辅助层，而不是独立的工具页。

## 用户痛点

- 信息密度高：主播口播、商品信息、优惠信息和评论同时出现，用户容易错过关键内容。
- 决策链路短：直播间购买决策往往发生在几分钟内，需要更快理解商品价值。
- 优惠规则复杂：价格、赠品、限时权益和组合优惠分散表达，用户需要二次整理。
- 评论滚动快：用户问题容易被淹没，也不知道如何组织更有效的提问。
- 个体适配难：美妆护肤商品需要结合肤质、使用场景和搭配方式判断是否适合。

## AI 设计机会点

RedLive AI 将 AI 能力嵌入直播间上下文，重点处理“理解、提炼、提问、决策”四类体验断点：

- 总结刚刚内容：帮助用户快速补齐错过的主播讲解。
- 提炼优惠信息：把分散的价格、赠品和规则整理成可扫读信息。
- 生成提问：根据用户关心的问题生成更具体、更容易被主播回应的提问。
- 判断是否适合我：围绕肤质、场景和购买风险给出轻量决策建议。

设计重点是让 AI 像直播间里的辅助助手，而不是打断观看流程的独立 AI 面板。

## 核心功能

- 移动端直播间画布：在桌面浏览器中以手机容器展示，模拟真实 App 截图氛围。
- 主播信息栏：展示主播头像、昵称、在线人数和关注状态。
- 直播主视觉：使用直播背景图作为 full-bleed 内容画面，保留讲解标题和商品说明。
- 评论弹幕：最多展示 4 条评论，支持发送新评论并进入弹幕列表。
- 右侧互动区：支持点赞、收藏、分享和 AI 入口。
- 商品浮层：展示水光修护精华、直播价、核心标签、赠品信息和“去看看”CTA。
- AI Bottom Sheet：底部抽屉展示 AI 直播助手、快捷操作和结构化结果。

## 关键交互

- 点赞：点击后心形变为小红书红 `#FF2442`，点赞数从 `2.7万` 变为 `2.8万`，再次点击恢复。
- 收藏：点击后收藏图标变为黄色 `#FDBC5F`，文案从“收藏”变为“已收藏”，再次点击恢复。
- 关注：点击后从红色“关注”切换为低噪声“已关注”，避免持续占用主 CTA 层级。
- 评论发送：底部输入评论后，新评论进入弹幕区，保持最多 4 条。
- AI 助手：点击右侧 AI 按钮后弹出 Bottom Sheet，默认展示“直播高光总结”。
- AI 快捷操作：切换“总结刚刚内容 / 提炼优惠信息 / 生成提问 / 判断是否适合我”，结果区域同步更新为标题 + 要点结构。

## 技术栈

- React
- Vite
- Tailwind CSS
- lucide-react
- 模拟数据
- Codex 辅助生成与迭代

Demo 不接真实后端、不接真实直播流，重点是验证移动端 AI 互动体验闭环。

## Vibe Coding 工作流

本项目采用 Vibe Coding 方式推进，用 AI 把产品想法快速转成可运行原型：

1. 定义项目 Brief：先明确直播间场景、用户痛点、AI 机会点和 Demo 范围。
2. 快速生成第一版：使用 Codex 初始化 React + Vite + Tailwind 项目，并搭建可运行直播间原型。
3. 体验方向重构：将最初偏 Dashboard 的界面重构为移动端直播间手机画布。
4. 设计诊断：引入 UI audit，识别视觉过重、红色过多、阴影过强、商品卡偏促销、AI 面板工具感强等问题。
5. 系统化收敛：基于 Xiaohongshu DESIGN.md 和 frontend-design skill，对颜色、字体、圆角、阴影、商品卡、弹幕和 Bottom Sheet 逐轮收敛。
6. 高保真微调：接入直播背景、主播头像和商品图素材，修复安全区、状态反馈和按钮层级。

这个流程的重点是：UX 设计师不只输出静态稿，而是用 AI 将需求理解、交互设计、视觉判断和前端 Demo 串成一个可展示、可点击、可迭代的作品。

## 设计系统接入方式

项目中引入了开源 Xiaohongshu 风格设计规范作为上下文：

- `docs/xiaohongshu-design-system.md`：开源 Xiaohongshu 风格规范，作为 UI 收敛参考。
- `docs/design-system-usage.md`：记录本项目后续 UI 修改的使用原则。
- `docs/ui-design-audit.md`：针对当前 Demo 的设计系统诊断与优化优先级。
- `.agents/skills/frontend-design`：前端设计 Codex Skill，用于辅助提升界面结构和视觉质量。

需要特别说明：本项目不是照搬小红书官方内部设计系统，而是参考开源 Xiaohongshu 风格规范，提炼出适合本 Demo 的设计原则：

- 内容优先，UI 退后
- 品牌红 `#FF2442` 克制使用
- 减少重阴影和厚玻璃拟态
- 使用移动端 Bottom Sheet 承载二级 AI 操作
- 商品卡保持轻量，不做强促销页面
- 直播画面保持生活方式内容感，而不是普通 AI 展示页

## 如何运行项目

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览构建结果：

```bash
npm run preview
```

## 后续可迭代方向

- 更真实的直播字幕输入：用模拟 transcript 驱动 AI 总结，而不是静态结果。
- 个性化适配：让用户选择肤质、预算和偏好后，生成更个性化的购买建议。
- 优惠规则解析：将券、满减、赠品和组合购拆成更清晰的决策卡片。
- 评论智能提问：根据当前弹幕热点生成更自然的问题建议。
- AI 结果可信度表达：补充“基于刚刚讲解 / 基于商品信息 / 需要向主播确认”的来源提示。
- 面试展示版本：整理设计过程截图、迭代前后对比和关键决策说明，形成完整作品集案例。

## 项目价值

RedLive AI 展示的是一种面向 AI Native 体验设计的工作方式：从场景问题出发，用 AI 快速搭建可运行原型，再通过设计系统和体验诊断持续收敛视觉与交互质量。

最终 Demo 从最初的“AI 展示页 / 促销 Demo”优化为更贴近小红书的“移动端直播内容场景”，更适合作为 UX 设计师在直播体验、AI 互动和 Vibe Coding 能力上的展示项目。
